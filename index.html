linux_常用命令总结

发表于 2020-08-02 更新于 2020-08-16 分类于 linux

GDB 学习

1. GDB 调试程序的三种方式（从应用场景出发）

  • gdb filename

    直接调试编译后生成的二进制文件

  • gdb attach pid

    调制某个正在运行的程序，通过任务管理命令（例如ps、top）获取pid

  • gdb filename corename

    core文件：一般在程序崩溃时会产生core文件

    linux系统一般默认不开启崩溃产生core文件这一机制，可以使用命令 ulimit 进行查看、修改等操作
